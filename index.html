How we built it

A system, not a prompt.

We treated Claude as an engine in the middle of a three-part pipeline — layered context, modular prompts, and a feedback loop that sharpened output over time.

01 Context

Persistent project inputs

ICP, messaging pillars, brand tone, and historical winning content — reused across every prompt so Claude stayed on voice.

02 Prompting

Modular prompt chains

Topics → score → outline → draft → critique. Five focused passes produced sharper posts than one monster prompt ever did.

03 Voice

Style transfer from real posts

"Rewrite this in the tone of a data-driven but conversational SaaS founder."
04 Self-Critique Loop

Draft → Critique → Rewrite

Claude reviewed its own output against clarity and insight criteria before anything left the system. Fewer platitudes, stronger hooks, tighter CTAs.

"Critique this for clarity and insight."
05 Distribution

LinkedIn as the listening post

Posts published via scheduling tools. Engagement triggered lead capture through LinkedIn forms and dedicated landing pages — routed straight into HubSpot.

06 Validation

HubSpot as the filter

Claude supported lead-scoring refinement, behavioral email personalization, and segmentation. Strategy-content engagers got different nurture tracks than tactical-content readers.

07 Optimization Loop · Critical

HubSpot data → fed back into Claude.

"Posts about ROI generated higher-quality leads — prioritize similar angles." Every two weeks, CRM signals reshaped the voice brief and the scoring rubric. The system got sharper without getting more expensive.

Key Insight

Claude's biggest impact came from structured thinking, not just content generation. Every output aligned with a downstream business outcome.

How we built it

Claude as the coach, not the dashboard.

Structured inputs. Constrained outputs. One green button per screen. The interface did less, so the affiliate could do more.

01 Data Input

Structured context, every time

Page content, performance metrics (CTR, conversions, bounce rate), and content metadata — all handed to Claude in a consistent shape.

02 Prompting

Constraints that shaped the output

"Identify the top 3 conversion blockers. Fix in simple language. Each must be actionable within 10 minutes."

Claude returned Problem → Fix → Expected Impact. Never vague. Never more than three.

03 Simplification

Plain English only

No jargon. No "optimize your on-page CRO signals." Claude wrote like a smart friend who doesn't do this for a living.

04 Model Selection

Haiku + Sonnet

Haiku for nightly recommendation volume. Sonnet for weekly long-form rewrites. Cost dropped; quality held.

05 Interface

One screen, three cards

Action cards, progress checklists, and Before/After examples. Expandable "advanced" options tanked engagement in testing — so we cut them.

06 Gamification

Points that mean something

Bonus points when a recommendation actually lifted conversion after 7 days. Monthly top-10 got a cash bonus + newsletter feature.

07 Continuous Learning

High-performing patterns reused. Low-impact recs removed.

Implementation + 7-day lift data flowed back into the prompt context. Network-wide learning without network-wide meetings.

🎯

Key Insight

The lift came from making action easy, not from being clever. Claude's job was to be clear, specific, and short — in that order.

Affiliates didn't need more tools. They needed a nudge.

Underperforming pages stayed underperforming because fixing them felt overwhelming. Churn was high because affiliates stopped seeing effort pay off. The network burned support hours answering "why isn't my site earning?"

  • Clarity over complexity
  • Action over insight
  • Behavior over data

ROI Driver 02

Final Takeaways

What both projects have in common.

Neither case study was about replacing humans — it was about removing the boring parts of the job. The pattern that worked in both, was the same.

01

Structured > ad hoc

Projects that used prompt frameworks, iterative loops, and context layering performed significantly better than one-off prompting.

02

Features > model size

The features that drove results:

  • Long-context memory
  • Iterative refinement
  • Role-based prompting
  • Constraint-based outputs
03

Feedback > fresh prompts

Both case studies improved performance by feeding real-world results back into prompts and continuously refining outputs.

Workflow Imagery Guide

How each system flows.

Seven steps per workflow. Two closed loops. Both end by feeding results back into Claude.

Workflow 01 — Thought Leadership Engine
Claude + LinkedIn + HubSpot

  1. Strategic Input Setup

    Define ICP, messaging, and goals. Feed into Claude as structured context.

  2. Content Ideation

    Generate topics → score → select. Align with business goals, not just engagement.

  3. Content Creation

    Draft → critique → refine loop. Output: LinkedIn posts + email content.

  4. Distribution (LinkedIn)

    Publish via personal or brand accounts. Drive engagement and inbound interest.

  5. Lead Capture (HubSpot)

    Capture via forms / CTAs. Score and segment based on behavior.

  6. Nurture & Conversion

    Personalized email sequences. Sales engagement only for qualified leads.

  7. Feedback Loop

    HubSpot data → back into Claude. Improve messaging, targeting, content strategy.
